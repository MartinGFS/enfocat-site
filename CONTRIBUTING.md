# Guía de contribución a Enfoca T

¡Gracias por tu interés en contribuir a **Enfoca T**!  
Este documento explica cómo puedes participar, reportar problemas y proponer mejoras.

---

## 🐞 Reporte de bugs
- Antes de abrir un nuevo issue, revisa el **[Meta‑Issue de seguimiento](./issues/#1)** para ver si tu problema ya está listado.
- Si tu bug ya está mencionado, comenta allí para aportar más detalles.
- Si es un bug nuevo, crea un issue independiente y enlázalo al meta‑issue con:  
  `Relacionado con #Meta-Issue-EnfocaT`.

---

## 🚀 Propuestas de mejora
- Usa la etiqueta `enhancement` para sugerir nuevas funcionalidades.
- Explica claramente el beneficio para los usuarios.
- Si tu idea ya aparece en el meta‑issue, comenta allí para sumar tu apoyo.

---

## 💬 Preguntas y soporte
- El canal oficial de soporte técnico es la sección **Issues de GitHub**.  
- Abre un issue con la etiqueta `question` si necesitas ayuda.  
- Para seguimiento general, consulta siempre el meta‑issue.

---

## 🔖 Etiquetas
Usamos las siguientes etiquetas para organizar los issues:
- `bug` → errores o fallos detectados.
- `enhancement` → nuevas funcionalidades o mejoras.
- `question` → dudas de uso o soporte.
- `documentation` → mejoras en la documentación.
- `discussion` → debates sobre ideas o cambios futuros.

---
